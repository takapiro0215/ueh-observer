CLI Output Format Spec v0.1

UEH Observer Board Mode (Aave v3 / Ethereum)

0. 目的

  • 画面は電光掲示板：短文・高密度・低遅延

  • ただし信頼の根拠（精度メタ）を必ず表示

  • 助言・最適化・実行はしない（状態のみ）

1. 出力の基本構造（1画面 = 1ウォレット）

固定順序（崩さない）

  1. Header（識別）

  2. Position Summary（主要指標）

  3. Trust Metadata（信頼メタ）

  4. Status Line（状態ラベルのみ）

  5. Footer（更新/次回）

2. 標準表示（NORMAL / STABLE）

表示例：

[UEH OBSERVER BOARD]  Aave v3 | Ethereum
Wallet: 0x7f3A...9C2D
HF: 1.23
LiqDist: 9.4%
Coll(USD): 12,483.21
Debt(USD): 8,972.10
LTV: 71.9%   LT: 79.5%
Block: 19,283,746  (lag 0)
OracleAge: 34s
Source: primary(ok) / secondary(ok)
Status: STABLE
LastUpdate: 12s ago   NextCheck: 15s

各項目の意味（短く固定）

  • HF: Health Factor（Aaveの定義）

  • LiqDist: 清算境界までの距離（%）

    • 注：算出式は内部だが、少なくとも「境界に近いほど小さい」単調性を保証

  • Coll/Debt: USD換算（Aaveの参照価格に準拠）

  • LTV / LT: Loan-to-Value / Liquidation Threshold

3. 境界接近（BOUNDARY）

境界域に入ったら「説明」ではなく、状態ラベルと検証強化の痕跡だけを出す。

表示例：

[UEH OBSERVER BOARD]  Aave v3 | Ethereum
Wallet: 0x7f3A...9C2D
HF: 1.17
LiqDist: 6.1%
Coll(USD): 12,201.08
Debt(USD): 10,114.55
LTV: 82.9%   LT: 84.0%
Block: 19,283,812  (lag 1)
OracleAge: 58s
Source: primary(ok) / secondary(ok)
Verify: STRICT (boundary zone)
Status: BOUNDARY_APPROACHING
LastUpdate: 7s ago    NextCheck: 10s

境界域ルール（表示人格を固定）

  • Verify: STRICT (boundary zone) を必ず出す

  • NextCheck を短くしてもOK（例：10s）

  • ただし助言文は出さない（例：「返済推奨」禁止）

4. 劣化表示（DEGRADED）

値は出すが、「信頼が落ちている」ことを先に伝える。
（電光掲示板で「情報収集中」と出すイメージ）

表示例：

[UEH OBSERVER BOARD]  Aave v3 | Ethereum
Wallet: 0x7f3A...9C2D
HF: 1.22
LiqDist: 9.0%
Block: 19,283,901  (lag 6)
OracleAge: 2m14s
Source: primary(ok) / secondary(timeout)
Status: DEGRADED
Reason: secondary_unavailable
LastUpdate: 21s ago   NextCheck: 30s

DEGRADED時の必須要件

  • Reason を1行で出す（短い原因コード）

  • Coll/Debt/LTV等の詳細は省略してよい（表示時間短縮のため）

5. 拒否（REFUSAL）

「出せないときは出さない」。これが信頼の根っこ。
REFUSALでは 指標を断言表示しない。

表示例：

[UEH OBSERVER BOARD]  Aave v3 | Ethereum
Wallet: 0x7f3A...9C2D
Status: REFUSAL
Reason: inconsistent_sources
primary_block: 19,283,910
secondary_block: 19,283,883
delta: 27 blocks
LastUpdate: 3s ago    NextCheck: 10s

REFUSALの鉄則

  • HFやLiqDistを出さない（誤断言防止）

  • Reason と最小限の根拠（ブロック差など）だけ出す

  • ここで“静かに強い”印象が出る

6. Status ラベル一覧（固定語彙）

表示は必ず Status: の1行で表す（UIの統一）。

  • STABLE

  • WATCH

  • BOUNDARY_APPROACHING

  • DEGRADED

  • REFUSAL

※ WATCH は「境界までは遠いが変動が増えている」など、将来導入枠。MVPでは省略可。

7. 原因コード（Reason）一覧（短く、機械可読）

DEGRADED / REFUSAL のReasonはこの語彙から選ぶ：

  • secondary_unavailable

  • primary_unavailable

  • timeout

  • rate_limited

  • inconsistent_sources

  • stale_oracle

  • unexpected_response

  • unsupported_asset

  • internal_error

8. コマンド体系（最小・電光掲示板向け）

Board（ライブ更新）

ueh board --chain eth --protocol aavev3 --wallet 0xABC... --interval 15s

Scan（1回だけ）

ueh scan --chain eth --protocol aavev3 --wallet 0xABC...

JSON（機械連携・ログ用）

ueh board --wallet 0xABC... --json

9. JSON出力（v0.1最小スキーマ）

※Board表示と同じ情報を、機械可読で吐く（Public repoに置いてOK）

{
  "chain": "ethereum",
  "protocol": "aavev3",
  "wallet": "0x...",
  "status": "STABLE",
  "metrics": {
    "health_factor": 1.23,
    "liq_distance_pct": 9.4,
    "collateral_usd": 12483.21,
    "debt_usd": 8972.10,
    "ltv_pct": 71.9,
    "liquidation_threshold_pct": 79.5
  },
  "trust": {
    "block_number": 19283746,
    "lag_blocks": 0,
    "oracle_age_sec": 34,
    "source": {
      "primary": "ok",
      "secondary": "ok"
    },
    "verify_mode": "NORMAL"
  },
  "timing": {
    "last_update_sec_ago": 12,
    "next_check_sec": 15
  },
  "reason": null
}

REFUSAL時は metrics を null にする：

{
  "status": "REFUSAL",
  "metrics": null,
  "reason": "inconsistent_sources"
}
